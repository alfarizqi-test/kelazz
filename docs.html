Integrasi Laravel Breeze dengan Spatie Laravel Permission

Panduan langkah demi langkah untuk menggabungkan autentikasi sederhana Laravel Breeze dengan sistem role & permission dari Spatie.

Kita gabungkan dua dunia yang sering bikin nyaman developer Laravel: Laravel Breeze untuk autentikasi sederhana, dan Spatie Laravel Permission untuk manajemen role & permission.

Kombinasi ini sangat populer untuk membangun aplikasi yang rapi tapi tetap fleksibel. Kita jalan pelan-pelan seperti resep masakan ilmiah üß™.

1. Mulai dari Proyek Laravel Bersih

composer create-project laravel/laravel nama-proyek cd nama-proyek

Siapkan database di file .env:

DB_DATABASE=nama_db DB_USERNAME=root DB_PASSWORD=

Lalu jalankan migrate:

php artisan migrate

BAGIAN A ‚Äì Install Laravel Breeze

Install Breeze:

composer require laravel/breeze --dev php artisan breeze:install

Pilih stack sesuai kebutuhan, misalnya Blade:

php artisan breeze:install blade npm install npm run dev php artisan migrate

Sampai tahap ini kamu sudah punya fitur: Register, Login, Logout, dan Dashboard.

BAGIAN B ‚Äì Install Spatie Laravel Permission

Install package:

composer require spatie/laravel-permission

Publish config dan migration:

php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"

Lalu migrate lagi:

php artisan migrate

BAGIAN C ‚Äì Integrasi ke User Model

Tambahkan trait berikut pada app/Models/User.php:


use Spatie\Permission\Traits\HasRoles;

class User extends Authenticatable
{
    use HasFactory, Notifiable, HasRoles;
}

BAGIAN D ‚Äì Membuat Role & Permission

Buat seeder:

php artisan make:seeder RolePermissionSeeder

Isi file seeder:


use Illuminate\Database\Seeder;
use Spatie\Permission\Models\Role;
use Spatie\Permission\Models\Permission;

class RolePermissionSeeder extends Seeder
{
    public function run()
    {
        Permission::create(['name' => 'manage users']);
        Permission::create(['name' => 'edit posts']);
        Permission::create(['name' => 'delete posts']);

        $admin = Role::create(['name' => 'admin']);
        $user  = Role::create(['name' => 'user']);

        $admin->givePermissionTo(Permission::all());
    }
}

Daftarkan di DatabaseSeeder lalu jalankan:

php artisan db:seed

BAGIAN E ‚Äì Integrasi dengan Breeze

Setiap user baru otomatis mendapat role user. Tambahkan kode ini pada RegisteredUserController: 


$user = User::create([
    'name' => $request->name,
    'email' => $request->email,
    'password' => Hash::make($request->password),
]);

$user->assignRole('user');

BAGIAN F ‚Äì Proteksi Route

Contoh proteksi berdasarkan role:


Route::get('/admin', function () {
    return view('admin.dashboard');
})->middleware(['auth', 'role:admin']);

Atau berdasarkan permission:


Route::get('/posts', function () {
    return view('posts');
})->middleware(['auth', 'permission:edit posts']);

BAGIAN G ‚Äì Blade Directive

Contoh pengecekan role di Blade:


@role('admin')
    Admin Panel
@endrole

Cek permission:


@can('edit posts')
    
@endcan

BAGIAN H ‚Äì Membuat Admin Manual


php artisan tinker

$user = App\Models\User::find(1);
$user->assignRole('admin');

Pola Mental

  • Breeze = pintu rumah (login system)
  • Spatie = penjaga ruangan (role & permission)
